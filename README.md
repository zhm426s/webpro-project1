# WP Project 1: Murder Mystery

This project is a HTML and CSS-only web game in which the user will attempt to solve one of two murder cases. It features CSS transitions and animations to provide a visually interesting, engaging experience. 

## Installation

Download all files and upload them to your server. 

If you would like to exclude items required for grading this project as an assignment for our class, do not download the "req" folder and erase everything in the footer of "index.html" using your preferred text editor. After completing that edit, then you can upload all files to your server.

## Usage

On every page, there are buttons in the header labeled "Switch Theme". Click this to switch between the light theme (default), and the dark theme.

Begin on the home page "index.html". Enter your preferred case file using either link on the home page. 

The case file page will display basic information about your selected case, as well as instructions on how to use the other pages. Use the in-text links or navigation bar links at the top to access the case board page, the suspects page, and the evidence page. 

The case board page contains a section meant to mimic a real-life detective's cork board. On it, there are pictures (labeled) with images of evidence pieces and suspects related to the case. Click on these to be referred to the suspect page or the corresponding section on the evidence page.  

Below the case board section, there are four hints. Click on each hint card to reveal the text of that hint, which includes further links to sections on the evidence page. 

The suspects page contains a lineup of suspects for the given case. Each suspect has a picture, a description, and an arrest button. Click the link in the arrest button to be directed to either the winning or losing page, depending on who the murderer was. 

The evidence page consists of a list of evidence pieces, each section dedicated to one piece and containing a picture, a description, a transcript (if applicable) which can be hidden or shown, any other notes, and a button to return to the case board. 

The winning and losing pages simply display a message saying whether or not the user won the game and arrested the correct suspect. They contain a button to return to the home page. 

The pages under the "req" folder are linked on the home page. They are requirements for this project to be graded. 

## Roadmap

In the future, we would like to add a Timer element to the site, which would keep track of how long it takes for a user to solve a mystery. 

Along with this we would also like to add a stats page, where users can keep track of their solved cases, their times, etc. This would likely require many more than two cases being included.

## Contributing

We are open to contributions, especially involving adding more cases to the game. Here is how to do add your own case: 

1. Duplicate all case related pages (case-file.html, case-board.html, suspects.html, evidence.html) and give them unique file names. 
2. Duplicate the "enter" button on the home page and link that to your new case file page. 
3. Design your mystery in any way you like. We suggest making a document to keep your ideas and image assets in while designing. 
4. Add your image assets to the corresponding subfolders of "images". 
5. Replace text and image content in your duplicate files with text and images related to your case.
